SUPABASE SETUP

1) Create a Supabase project.
2) Open SQL Editor and run supabase_schema.sql
3) Open Project Settings / Connect and copy:
   - Project URL
   - anon / publishable key
4) Open app.js and replace:
   - PASTE_YOUR_SUPABASE_URL_HERE
   - PASTE_YOUR_SUPABASE_ANON_KEY_HERE
5) Deploy index.html, style.css, app.js, and _redirects to Netlify.
6) Open the site. On first load, the app will auto-bootstrap the default data into Supabase.

IMPORTANT SECURITY NOTE
This version keeps your current in-app login and does NOT use Supabase Auth yet.
Because of that, the SQL creates anon policies that allow browser CRUD access.
This is convenient for launch/testing, but it is not hospital-grade security.
For production, the next step should be migrating users and roles to Supabase Auth + stricter RLS.
